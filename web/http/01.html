# 网络协议

# 一、HTTP 版本

# HTTP/0.9(1991)

  • 已定义协议最核心内容:
    • C/S 基本结构.
    • 用 "域名 or IP+端口" 确定目标地址.
    • 求方法只支持 GET.
    • 一行请求(The One-line Protocol): GET /target.html
  • 实践:
    • 通过 telnet 请求 Google 页面.(Tip: 0.9版本协议不带版本号)

# HTTP/1.0

  • 第一个具备广泛实际应用价值的协议版本:
    • 引入返回码、header、多字符集、多行请求
    • 请求方法只支持 GET、HEAD、POST.
  • 实践:
    • 通过 netcat 发送多行请求
    • Tip: 通过改 UA 头模拟不同浏览器，常用语分析浏览器适配.

# HTTP/1.1

  • 最广泛使用的协议:
    • 大量帮助传输效率提升的特性被加入（RFC2616)
    • 长连接: 解决 HTTP/1.0 版本每组请求和响应交互都要完成一次 TCP 的链接和关闭操作的问题.
  • 实践:
    • 方式: netcat
    • 发送请求: 使用 Connection: keep-alive 开启长链接的 HTTP 头.
    • 响应: HTTP 头部的 Transfer-Encoding: chunked 表示采用分块传输编码.(极大提高了灵活性, 在)
      • 开头指的是接下来这一块的大小
      • 响应信息
      • 在有意义的块传输完成之后，会紧跟上一个长度为 0 的块和一个空行，代表传输结束
      • (Optional) 代表结束的 0 之后，协议允许放一些额外信息，称为"Trailer"是用于校验正确性的 checksum.(数字签名，传输完成状态等)
    • Tip: 长连接开启的情况下，必须使用 Content-Length 或者 chunked头的一种.
