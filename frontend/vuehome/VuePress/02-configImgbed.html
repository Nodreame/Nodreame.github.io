# 为什么要写这篇博客

前几天看到松哥的文章 -- 写博客必备！手把手教大家搭建免费图床，真香！，七牛云支持免费的 10G 空间可以用来搭建个人图床，看完直接真香果断回坑，从此开始超爽的写文之旅.

松哥推荐了 PicGo 作为客户端工作, 我调查了一下发现这个插件居然有 VSCode 的插件版本 vs-picgo, 一看简介发现还支持剪切板上传！搜索了下网上暂时没有相关的文章我就写一篇安利大家吧~

# 为什么要用图床

为什么写博客要用图床？简单分析了下原因如下:

  • 1、博客图片加入 md 文件步骤繁琐 & 包含图片导致工程体积过大
  • 2、步骤繁琐，平日在 VSCode 用 Markdown 写文章，插入图片的步骤是：
    • 截图: Shift+Cmd+4 截图，右键存放到桌面
    • 调整: 用 "Skitch" 做箭头 | 文字 | 画框标记
    • 改名: 改成规范名称
    • 移动: Ctrl+` 开启 VSCode 终端，用 mv ~/Desktop/xx.jpg ./ 移动
    • 填入: 按照 Markdown 规范编写链接

如果用了图床呢？可以分成两种情况:

  • 无需调整: Ctrl+Shift+Options+4 截图到剪切板, 直接在 Markdown 文档目标位置 Options+Cmd+U 上传.

  • 需调整情况:
    • 截图: Shift+Cmd+4 截图，右键存放到桌面
    • 调整: 用 "Skitch" 做箭头 | 文字 | 画框标记
    • 上传: 回到 Markdown 文档，Options+Cmd+E 选择桌面调整好的文件上传.

Amazing!
